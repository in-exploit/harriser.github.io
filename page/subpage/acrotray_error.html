禁用Adobe AcroTray.exe从启动时启动

2019年9月13日

问题描述

系统运行一段时间后，文件或文件夹单击右键时，经常假死状态，其他功能正常

查看资源管理器发现发现，发现AcroTray.exe，结束进程并将文件修改后系统频繁报错，重启后依旧

右键单击文件或文件夹时，（Windows上下文菜单）会调用AcroTray.exe。它负责上下文菜单中的Adobe相关条目。

提示

禁用或删除AcroTray.exe，可能会导致部分程序调用Acrobat时失败；

在禁用服务和启动项完成前删除或修改文件，可能会导致AcroTray错误提醒频繁弹出，可不关闭对话框，禁用服务和启动项后，不会再频繁弹出。

处理

  1. 打开资源管理器"服务(Services.msc)"页，停止以下服务，设置为手动启动。

    Adobe Acrobat Update Adobe Genuine Software Integrity Adobe Genuine Software Integrity Service

  2. 下载 Autoruns 实用程序，在此处禁用所有Adobe相关程序

    Adobe Updater Startup Utility Acrobat Assistant（AcroTray） Adobe Acrobat Synchronizer

  3. 下载 ShellExView 。

    打开 ShellExView，选择"选项"并勾选"显示32位扩展"，查找并禁用以下选项。

    Shell Ex扩展。现在禁用，和。现在在选项中取消选择显示32位Shell扩展并再次禁用上述项目以及。

    Adobe Acrobat Create PDF from Selection Adobe Acrobat Create PDF Helper Adobe Acrobat Create PDF Toolbar

    取消"显示32位扩展"，检查并禁用上述项目，以及禁用以下项目

    Acrobat Elements Context Menu

  4. 从任务管理器中结束AcroTray.exe，注销并登录到Windows，打开任务管理器查看AcroTray.exe启动状态。

    打开.pdf文件，查看AcroTray.exe启动状态

    如果AcroTray.exe不再启动，这样你就成功了！Congratulation

  5. 运行任务管理器，检查Adobe Tasks（如果有）并禁用它们。
